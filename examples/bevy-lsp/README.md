# My ggen Project

Generated by `ggen init`.

## Getting Started

1. **Initialize project**: Run `make setup` to prepare your environment
2. **Define schema**: Edit `schema/domain.ttl` with your domain model
3. **Create templates**: Add Tera templates in `templates/` for your target languages
4. **Generate code**: Run `make build` to generate code from your ontology

## MCP/A2A Configuration

This project includes MCP (Model Context Protocol) and A2A (Agent-to-Agent) configuration files:

- `.mcp.json` - MCP server configuration
- `a2a.toml` - A2A agent configuration

To reinitialize these configs:
```bash
ggen mcp init-config --force
```

To validate configuration:
```bash
ggen mcp validate-config
```

To set up Claude Desktop integration:
```bash
ggen mcp setup
```
